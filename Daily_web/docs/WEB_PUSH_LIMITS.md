# Ограничения веб-уведомлений для Daily Web

Этот файл нужен, чтобы веб-реализация не обещала поведение, которое веб-платформа не гарантирует.

## Что реально можно требовать

- Данные приложения можно надежно хранить локально на устройстве через `IndexedDB`.
- PWA можно установить на Android как приложение с иконкой на домашний экран.
- На iPhone web push работает не как у обычного сайта во всех режимах: нужен установленный на экран "Домой" веб-app/PWA и разрешение пользователя.
- Для фоновых уведомлений после закрытия приложения нужен `Service Worker` и `Push API`.
- Для точных напоминаний по расписанию нужен серверный scheduler, потому что чисто клиентский web не гарантирует локальные уведомления по времени после полного закрытия браузера.

## Что нельзя обещать без backend

- Надежные уведомления по расписанию только силами фронтенда.
- 100% идентичное поведение native Android scheduler на всех браузерах.
- Гарантированную доставку reminder'ов при убитом браузере без server push.

## Рекомендуемая архитектура

- Клиент хранит задачи, историю и настройки в `IndexedDB`.
- Клиент регистрирует `PushSubscription`.
- Клиент отправляет изменения расписания на backend.
- Backend хранит напоминания в UTC, учитывает timezone пользователя и отправляет web push в нужное время.
- `Service Worker` показывает уведомление и открывает нужный экран/задачу.
