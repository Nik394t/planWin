# Feature Parity: Mobile Daily -> Web Daily

Ниже зафиксирован минимум, который веб-версия должна повторить по текущему мобильному приложению `Daily`.

## Навигация

Нижняя навигация:
- Планы
- Молитвы
- История
- Настройки

## Экран "Планы"

- Периоды: `День`, `Неделя`, `Месяц`, `Год`
- Переключение периода внутри одного экрана
- Список задач для выбранного периода
- Создание задачи
- Редактирование задачи
- Удаление задачи
- Отметка выполнения
- Замочек/закрепление задачи
- Дневной прогресс отдельно от остальных прогрессов
- Автопереход на новый день при rollover даты
- Блокировка редактирования завершенных или прошедших периодов

## Задачи

Поля задачи:
- `id`
- `scope`: `day | week | month | year`
- `periodKey`
- `title`
- `description`
- `source`: `manual | recurring | prayer | prayerDefault`
- `isLocked`
- `recurringId`
- `isDone`
- `doneAt`
- `remindersEnabled`
- `reminders[]`
- `createdAt`
- `updatedAt`

## Напоминания по задаче

- Напоминания опциональны
- По умолчанию выключены
- Для дневной задачи выбирается только время в рамках текущего дня
- Для недельной/месячной/годовой задачи выбираются дата и время внутри периода
- Можно добавлять несколько напоминаний
- Прошедшее время выбирать нельзя
- Текст уведомления: `Напоминаю о задаче: <название задачи>`
- Для задач с замочком напоминание сохраняется на следующий релевантный период
- После выполнения задачи напоминания больше не приходят
- После завершения периода старые напоминания больше не приходят

## Экран "Молитвы"

- План по дням недели
- Для каждого дня можно задать молитвенную задачу
- Дневная молитва автоподставляется в дневной план
- Задачи молитвенного плана не ломают общую логику периодов

## Экран "История"

- История действий по дням
- Выбор конкретного дня
- Показ действий только за выбранный день
- Хранение событий: создание, редактирование, удаление, выполнение, открепление/закрепление, обновление настроек, итоги периода

## Экран "Настройки"

- Глобальный переключатель уведомлений
- Утреннее уведомление: on/off + время
- Вечернее уведомление: on/off + время
- Часовой пояс
- Тест уведомления
- Кнопка `О приложении`

## Экран "О приложении"

- Отдельный раздел с описанием, для чего нужен Daily
- Без упоминаний переноса из бота

## Данные и сохранение

- Сохранение на устройстве после закрытия приложения
- Данные не теряются при обновлении версии веб-приложения
- Для обновлений обязательны миграции схемы хранилища
- Уведомления после обновления должны восстанавливаться
